# Telegram Join Gate Bot

Бот для Telegram-групп и супергрупп.  
Запрещает новым (и неизвестным) участникам писать до тех пор, пока они вручную не добавят заданное количество новых людей. Переходы по ссылкам и QR-кодам **не засчитываются** — только ручное добавление через Telegram.

---

## Установка

```bash
git clone <repo-url>
cd bot
python3 -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

---

## Настройка

Создайте файл `.env` на основе `.env.example`:

```bash
cp .env.example .env
```

Отредактируйте `.env`:

| Переменная | Обязательна | Описание |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен бота от @BotFather |
| `DB_PATH` | ❌ | Путь к SQLite-файлу (по умолчанию `bot.db`) |
| `DEFAULT_REQUIRED_MEMBERS` | ❌ | Количество участников по умолчанию (по умолчанию `1`) |
| `WARNING_COOLDOWN_SECONDS` | ❌ | Интервал предупреждений в секундах (по умолчанию `10`) |

## Команды администратора

Все команды работают **только в группах** и **только для администраторов**.

| Команда | Описание |
|---|---|
| `/help` | Список команд |
| `/config` | Текущие настройки чата |
| `/setrequired <число>` | Установить количество участников для разблокировки |
| `/settopic` | Привязать бота к текущей теме (для форумов) |
| `/settopic <id>` | Привязать бота к теме по ID |
| `/settopic off` | Отключить привязку к теме |
| `/reloadtexts` | Применить изменения в `texts.py` без перезапуска |

## Необходимые права бота

В настройках группы выдайте боту следующие права:

- **Delete messages** — для удаления сообщений заблокированных пользователей
- **Restrict members** — для ограничения прав на отправку сообщений
- **Read messages** — для отслеживания входящих сообщений

Дополнительно в BotFather включите:

- `Allow Groups` → включить
- Установите **Group Privacy = disabled** (`/setprivacy` → `Disable`), чтобы бот получал все сообщения группы

## Структура проекта

```
bot/
├── main.py               # Точка входа: инициализация Bot, Dispatcher, роутеров
├── config.py             # Загрузка настроек из .env
├── database.py           # Асинхронный слой работы с SQLite (aiosqlite)
├── texts.py              # Все пользовательские тексты (редактируйте здесь)
│
├── handlers/
│   ├── admin.py          # Команды администратора (/help, /config, /setrequired, ...)
│   ├── members.py        # Вступление/выход участников, отслеживание ручных добавлений
│   └── messages.py       # Блокировка сообщений неразблокированных пользователей
│
└── services/
    ├── permissions.py    # restrict_user / unrestrict_user через Bot API
    └── settings.py       # Получение/создание настроек чата с дефолтами
```

## Логика работы

1. **Новый участник вступает** → бот приветствует, ограничивает права, сохраняет в БД.
2. **Пользователь вручную добавляет кого-то** → Telegram присылает service message с `from ≠ new_member`; бот инкрементирует счётчик.
3. **Счётчик достиг лимита** → бот снимает ограничения, отправляет поздравление.
4. **Пользователь пишет без доступа** → сообщение удаляется, бот предупреждает (не чаще раза в `WARNING_COOLDOWN_SECONDS` секунд).
5. **Пользователь выходит** → прогресс сбрасывается; при повторном вступлении — всё заново.
6. **Пользователь был в группе до добавления бота** → при первом сообщении ограничивается и получает предупреждение.

## Кастомизация текстов

Все сообщения находятся в `texts.py`. После редактирования выполните `/reloadtexts` в группе — перезапуск не нужен.

Доступные плейсхолдеры: `{username}`, `{required}`, `{added}`, `{remaining}`, `{count}`.
